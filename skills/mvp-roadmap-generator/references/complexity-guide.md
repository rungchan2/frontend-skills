# 기능 복잡도 및 소요시간 가이드

## 복잡도 기준표

### 프론트엔드 태스크

| 복잡도 | 예상 시간 | 예시 |
|--------|----------|------|
| **XS** | 0.5일 | 버튼 스타일 수정, 텍스트 변경, 간단한 CSS 조정 |
| **S** | 1일 | 정적 UI 컴포넌트, 단순 폼, 리스트 렌더링 |
| **M** | 2-3일 | CRUD 페이지, 폼 밸리데이션, 기본 상태관리 |
| **L** | 1주 | 복잡한 폼 위저드, 데이터 시각화, 실시간 기능 |
| **XL** | 2주+ | 에디터, 대시보드, 복잡한 인터랙션 |

### 백엔드 태스크

| 복잡도 | 예상 시간 | 예시 |
|--------|----------|------|
| **XS** | 0.5일 | 단순 엔드포인트 추가, 필드 추가 |
| **S** | 1일 | 기본 CRUD API, 단순 쿼리 |
| **M** | 2-3일 | 복잡한 비즈니스 로직, 트랜잭션 처리 |

| **XL** | 2주+ | 결제 시스템, 실시간 동기화, 복잡한 권한 |

### 풀스택 태스크 (프론트 + 백엔드)

| 복잡도 | 예상 시간 | 예시 |
|--------|----------|------|

## 의존성 패턴

### 일반적인 의존성 체인

```
프로젝트 설정 (T001)
    │
    ├── 인증 시스템 (F001)
    │       │
    │       ├── 프로필 관리 (F002)
    │       │
    │       └── 권한 관리 (F003)
    │               │
    │               └── 어드민 대시보드 (F004)
    │
    └── 핵심 CRUD (F005)
            │
            ├── 검색/필터 (F006)
            │
            ├── 댓글/좋아요 (F007)
            │
            └── 알림 (F008)
```

### 의존성 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| **Hard** | 필수 선행. 없으면 불가 | 인증 → 프로필 |
| **Soft** | 있으면 좋음. 없어도 가능 | 검색 → 자동완성 |
| **External** | 외부 서비스 의존 | 결제 → PG사 계약 |

## 버퍼 계산

### 버퍼 비율 가이드

| 상황 | 버퍼 | 이유 |
|------|------|------|
| 익숙한 기술 | +20% | 예상치 못한 이슈 |
| 새로운 기술 | +50% | 학습 곡선 |
| 외부 연동 | +30-50% | API 문서 불일치, 응답 지연 |
| 불명확한 요구사항 | +50-100% | 변경 가능성 |

### 예시 계산

```
기능: 결제 연동
- 기본 추정: 5일
- 새로운 PG사: +50% = 2.5일
- 요구사항 변경 여지: +30% = 1.5일
- 최종 예상: 9일 (2주 내)
```

## 마일스톤별 권장 범위

### M1: Alpha (핵심 기능)

- 기간: 4-6주
- 범위:
  - 인증 시스템
  - 핵심 CRUD 1-2개
  - 기본 UI/UX
- 목표: "동작하는 프로토타입"

### M2: Beta (확장 기능)

- 기간: 4-6주
- 범위:
  - 나머지 주요 기능
  - 검색/필터
  - 알림
  - 결제 (필요시)
- 목표: "테스트 가능한 제품"

### M3: Release (출시)

- 기간: 2-4주
- 범위:
  - 버그 수정
  - 성능 최적화
  - 모니터링
  - 문서화
- 목표: "출시 가능한 제품"
