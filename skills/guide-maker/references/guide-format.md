## 문서 구조

```
1. 페이지 위치 (Navigation path)
2. 기능 개요 (Feature overview with bullet list)
3. Part별 주요 기능 섹션 (Step-by-step guides)
4. 테이블 컬럼 설명 (Table column reference)
5. 자주 묻는 질문 (FAQ with toggles)
```

## 1. 페이지 위치

문서 시작 부분에 해당 기능 페이지로 이동하는 경로를 명시.

**구성요소:**
- H2 헤딩: "페이지 위치"
- 단락: **[상위 메뉴] → [해당 메뉴]** (굵게)
- 이미지 플레이스홀더 콜아웃
- 구분선

---

## 2. 기능 개요

페이지/기능의 전체적인 기능 목록을 불릿 리스트로 정리.

**구성요소:**
- H2 헤딩: "[기능명] 개요"
- 단락: "[기능명] 페이지에서는 다음 기능을 사용할 수 있습니다:"
- 불릿 리스트: **[기능명]**: [설명] 형식
- 구분선

---

## 3. Part별 Step 가이드

기능을 논리적 그룹(Part)으로 나누고, 각 Part 내에서 Step별로 설명.

### Part 헤딩
- H2: "Part N: [파트 제목]"

### Step 구성
- H3: "Step N. [단계 제목]"
- 단락: 단계에 대한 간단한 설명 1-2문장
- 이미지 플레이스홀더 콜아웃
- 번호 리스트: 실제 동작 단계
- (선택) 테이블: 버튼/옵션 설명
- (선택) 콜아웃: 팁, 주의사항
- 구분선

---

## 4. 테이블 컬럼 설명

데이터 테이블이 있는 페이지에서 각 컬럼의 의미를 설명.

**구성요소:**
- H2 헤딩: "테이블 컬럼 설명"
- 단락: "[테이블명]의 각 컬럼을 설명합니다."
- 테이블: 컬럼 | 설명
- 구분선

---

## 5. 자주 묻는 질문

토글 형식으로 FAQ 작성.

**작성 원칙:**
- 질문은 사용자 관점에서 작성 (존칭 사용)
- 답변은 구체적이고 실행 가능하게
- 관련 메뉴/버튼명은 **굵게** 표시
- 트러블슈팅 질문 포함

## 콜아웃 사용 가이드

### 용도별 아이콘/색상

| 용도 | 아이콘 | 색상 | 사용 예시 |
|------|--------|------|----------|
| 팁/도움말 | 💡 | blue_background | 사용 팁, 추천 방법 |
| 주의/경고 | ⚠️ | yellow_background | 비용 발생, 제한사항 |
| 중요/위험 | 🚨 | red_background | 삭제, 되돌릴 수 없는 작업 |
| 빠른팁 | ⚡ | green_background | 효율적인 방법 |
| 알림/메시지 | 📱 | green_background | 알림톡, 메시지 관련 |
| 시간/일정 | ⏰ | yellow_background | 시간 제한, 자동 처리 |
| 이미지 | 📸 | gray_background | 스크린샷 플레이스홀더 |

### 사용 원칙
- Step당 최대 2개 권장
- 정말 중요하거나 강조할 내용에만 사용
- 같은 색상 콜아웃이 연속되지 않도록 배치

## 예시: 출결관리 가이드

### 페이지 위치 섹션
```
[H2] 페이지 위치
[단락] **가맹점 대시보드 → 출결관리**
[콜아웃 📸 gray] 스크린샷: 좌측 사이드바에서 '출결관리' 메뉴 위치를 표시한 화면
[구분선]
```

### 기능 개요 섹션
```
[H2] 출결관리 개요
[단락] 출결관리 페이지에서는 다음 기능을 사용할 수 있습니다:
[불릿] **반별 출결 관리**: 반과 수업 세션을 선택하여 학생별 출결 체크
[불릿] **출석 상태 버튼**: 출석, 비인정결석, 인정결석, 지각, 조퇴, 보강 원클릭 처리
[불릿] **학부모 알림톡 자동 발송**: 출석/비인정결석/지각 버튼 클릭 시 학부모에게 알림톡 자동 발송
[구분선]
```

### Step 섹션
```
[H2] Part 1: 기본 출결 관리
[H3] Step 1. 출결관리 페이지로 이동하기
[단락] 좌측 메뉴에서 출결관리 페이지로 이동합니다.
[콜아웃 📸 gray] 스크린샷: 출결관리 페이지 전체 화면
[번호1] 좌측 메뉴에서 **'출결관리'**를 클릭합니다
[번호2] 출결 관리 화면이 나타납니다
[구분선]
```

### 버튼 테이블 섹션
```
[H3] Step 5. 개별 학생 출결 체크하기
[단락] 각 학생의 출결 상태를 버튼 클릭으로 처리합니다.
[테이블]
| 버튼 | 색상 | 설명 | 알림톡 |
| **출석** | 초록색 | 정상 출석 처리 | ✅ 발송 |
| **비인정결석** | 빨간색 | 사유 없는 결석 | ✅ 발송 |
| **인정결석** | 회색 | 사유 있는 결석 | ❌ 발송 안 함 |

[콜아웃 💡 blue] **Tip**: 알림톡이 발송되는 버튼에 마우스를 올리면 미리보기를 확인할 수 있습니다.
[구분선]
```

### FAQ 섹션
```
[H2] 자주 묻는 질문
[토글] Q: 출석 버튼을 눌렀는데 알림톡이 안 갔어요
  [단락] A: 학생의 연락처가 등록되어 있는지 확인해주세요. **'학생등록'** 메뉴에서 학부모 연락처를 등록해야 알림톡이 발송됩니다.

[토글] Q: 알림톡 발송을 끄고 싶어요
  [단락] A: 출결관리 페이지 헤더의 **'알림 ON'** 버튼을 클릭하고 토글을 OFF로 변경하세요.
```

## 작성 시 주의사항

1. **일관성 유지**: 같은 유형의 정보는 같은 형식으로
2. **사용자 관점**: 기술 용어보다 이해하기 쉬운 용어
3. **정중한 어투**: ~합니다, ~됩니다
4. **구조화**: Part → Step 계층 유지
5. **이미지 플레이스홀더**: 모든 Step에 포함
